# Phase 1: セキュリティ改善

## 依頼概要

セキュリティ診断の結果、本アプリケーションに **2件** の脆弱性が報告されました。
優先度が高いため、早急に修正をお願いします。

---

## 1-A. SQLクエリの安全性

**対象**: `backend/src/routes/api.ts`

SQL文の組み立てにユーザー入力を直接埋め込んでいる箇所があります。
Prepared Statement（プレースホルダー）を使用する形に修正してください。

**レビュー観点**:
- `db.prepare()` に渡されるSQL文に、文字列結合やテンプレートリテラルでユーザー入力が埋め込まれていないか
- `?` プレースホルダーを使用し、値をパラメータとして渡しているか

---

## 1-B. ユーザー入力の表示方法

**対象**: `frontend/src/views/AdminPanel.vue`, `frontend/src/views/LotteryResult.vue`

ユーザーが入力したデータの表示方法に問題があります。
HTMLとしてそのまま描画される実装を、安全な方法に修正してください。

**レビュー観点**:
- Vue.js テンプレートで `v-html` が使われている箇所がないか
- ユーザー入力の表示には `{{ }}` (マスタッシュ構文) を使用しているか
- コメントの内容が適切か
